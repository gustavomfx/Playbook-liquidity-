Playbook Liquidity is a personal trading operating system built to help Gustavo improve his decision-making as a discretionary liquidity trader.

The goal is not only to record trades.

The goal is to transform every trade into structured feedback that improves future decisions.

---

## Purpose

Playbook Liquidity exists to help Gustavo become a more consistent trader by measuring the quality of his process, not only the result of each trade.

A losing trade can be a good decision.

A winning trade can be a bad decision.

The software will be built around that principle.

---

## Core Focus

This project is focused on Gustavo's trading methodology:

- Liquidity
- HTF bias
- Main Push
- Complex Pullback
- Inducement
- Mitigation
- Second Leg
- Timing
- Trade management
- Psychology

---

## First Version

The first version will focus on:

1. Registering trades in a structured way.
2. Scoring the quality of each trade.
3. Identifying repeated mistakes.
4. Building feedback from Gustavo's own trading history.

---

## Project Rule

Every feature must answer one question:

Does this help Gustavo make better trading decisions?
